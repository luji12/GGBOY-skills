# 分析专家 Skill

## 分析框架

### 数据分析流程
1. **理解问题**：分析目的是什么？决策者需要什么？
2. **数据收集**：数据来源、时间范围、样本量
3. **数据清洗**：缺失值、异常值、格式统一
4. **探索分析**：分布、趋势、相关性
5. **深度分析**：假设检验、因果推断
6. **洞察提炼**：关键发现 + 可操作建议
7. **可视化**：图表选择原则（趋势→折线、对比→柱状、占比→饼图）

### 研究报告结构
- **摘要**（300字以内，可独立阅读）
- **背景**：为什么做这个分析？
- **方法**：如何收集和分析数据？
- **发现**：数据说明了什么？
- **解读**：为什么会这样？
- **建议**：应该怎么做？
- **局限性**：分析的不足之处

### 上下文工程（Context Engineering）
处理长文本/大量信息时：
1. **分层摘要**：先摘要再细读
2. **关键词索引**：标记重要概念位置
3. **相关性排序**：按与任务的关联度排序
4. **矛盾识别**：找出信息间的冲突

## 分析思维工具

### MECE原则
相互独立、完全穷尽——分类不重叠不遗漏

### 第一性原理
从基本事实出发推导，而非类比现有方案

### 5 Why分析
连续问5次"为什么"找到根本原因

### 结论先行（金字塔原理）
结论 → 论据1/2/3 → 支撑事实
