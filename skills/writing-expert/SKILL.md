# 写作专家 Skill

## 写作类型与对应方法

### 技术文档
结构：概述 → 安装 → 快速开始 → 详细说明 → API参考 → 常见问题

原则：
- 用主动语态（"运行命令" 而非 "命令被运行"）
- 代码示例必须可运行
- 每个步骤只做一件事
- 提供预期输出

### 报告/分析文档
结构：执行摘要 → 背景 → 发现 → 分析 → 结论 → 建议

原则：
- 摘要放最前（读者可能只看这里）
- 数据支撑观点
- 结论可操作

### 故事/脚本写作
结构分析维度：
- **叙事弧线**：建立世界 → 冲突升级 → 高潮 → 结局
- **人物弧线**：初始状态 → 触发事件 → 变化历程 → 最终状态
- **主题强化**：每个场景都应服务于核心主题

### 文案写作
- 标题：5W1H中选1-2个，制造悬念或共鸣
- 正文：问题 → 解决方案 → 证明 → 行动召唤
- 语气：根据品牌调性调整（专业/亲切/幽默）

## 质量检查

- [ ] 读者是谁？语言是否适配？
- [ ] 逻辑是否连贯？有无跳跃？
- [ ] 关键信息是否突出？
- [ ] 有无错别字/语法错误？
- [ ] 长句是否可以拆短？
